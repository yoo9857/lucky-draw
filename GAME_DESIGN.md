# 오늘의 행운 (lucky-draw) — 게임 설계

> 앱인토스 미니게임. **하루 한 번 무료 행운 뽑기.**
> 설계 제1원칙: **사행성(도박) 요소 0 → 심사 통과 + 합법.**

---

## 1. 컨셉

매일 1회, **무료로** 행운을 뽑는다. 등급(꽝~잭팟)에 따라 **행운 포인트**(게임 내 재미 요소, 현금 환전 불가)를 얻고, 연속 출석으로 보너스를 받으며, 누적 포인트로 랭킹을 겨룬다.

핵심 재미 루프:
```
매일 접속 → 무료 뽑기(긴장감/연출) → 등급·포인트 획득 → 연속출석 보너스 → 랭킹 경쟁 → 내일 또 접속
```

---

## 2. 사행성 회피 설계 (가장 중요)

앱인토스 출시 심사에서 막히는 1순위가 사행성이다. 다음 4가지를 **구조적으로** 보장한다.

| 원칙 | 구현 | 효과 |
|------|------|------|
| **참가비 0** | 뽑기 비용 없음, 인앱결제와 추첨 분리 | "돈 내고 베팅" 구조 자체가 없음 |
| **환전 불가** | 보상은 게임 내 포인트/뱃지뿐, 현금·상품 교환 없음 | 사행행위 정의에 해당 안 됨 |
| **확률 공개** | `odds.ts` 단일소스 → 확률안내 화면에 그대로 노출 | 확률형 아이템 투명성 원칙 준수 |
| **과몰입 방지** | 하루 1회 제한(로컬 + 서버) | 반복 결제·중독 유발 없음 |

> ⚠️ "로또/복권" 단어는 이름·UI에서 배제(국가 독점어). "행운/럭키/뽑기"로 표기.

---

## 3. 게임 메커닉

### 3.1 확률표 (단일 진실 공급원 = `src/game/odds.ts`)
| 등급 | 확률 | 행운 포인트 |
|------|-----:|-----:|
| 🏆 잭팟(1등) | 0.1% | 1,000 |
| 🥇 2등 | 1.0% | 300 |
| 🥈 3등 | 5.0% | 100 |
| 🥉 4등 | 20.0% | 30 |
| 🎟️ 5등 | 40.0% | 10 |
| 💨 꽝 | 33.9% | 0 |

합계 100.0%. **이 표가 추첨 로직과 확률 고지 화면 둘 다를 구동** → 표시 확률과 실제 확률이 영원히 일치(불일치 사기 원천 차단).

### 3.2 연속 출석(streak) 보너스
- 연속 N일 접속 시 획득 포인트에 보너스 배율: `1 + min(N, 7) * 0.1` (최대 +70%)
- 하루 거르면 streak 1로 리셋

### 3.3 일일 1회 제한
- 같은 날(자정 기준) 이미 뽑았으면 버튼 비활성 + 다음 뽑기까지 카운트다운

## 4. 화면 구성

| 경로 | 화면 | MVP |
|------|------|:---:|
| `/` | 홈: 오늘 뽑기 버튼, 연속출석, 보유 포인트, 오늘 결과 | ✅ |
| `/result` | 결과: 뽑기 연출 + 등급/포인트 (MVP는 홈 내 모달로 통합 가능) | ✅ |

| `/ranking` | 누적 포인트 리더보드 (앱인토스 게임 리더보드 연동) | v2 |

## 5. 아키텍처

```
src/
  game/
    odds.ts       # 확률표 + 등급 정의 (단일소스)
    draw.ts       # 가중 랜덤 추첨 — 순수함수(rng 주입 → 테스트 가능)
    streak.ts     # 연속출석 계산 (날짜 비교, 보너스 배율)
    storage.ts    # 영속화 추상화 (일일제한·기록·포인트)
  pages/
    index.tsx     # 홈
    odds.tsx      # 확률 안내
pages/            # 라우터 재export 스텁 (자동 규칙)
```

설계 결정:
- **추첨은 순수함수 + RNG 주입** → 시드 고정으로 결정적 테스트 가능, 확률 검증 가능.
- **storage는 인터페이스로 추상화** → MVP는 인메모리/AsyncStorage, 운영은 앱인토스 storage SDK(with-storage 예시)로 교체. 상위 코드 영향 없음(스펙 격리, 증권 시스템과 같은 원칙).
- **MVP는 클라이언트 RNG**(무료·무환전이라 조작 위험 낮음). **보상에 가치가 생기는 v2**(랭킹 경쟁/리워드)에는 **서버 권위적 RNG + 일일제한 서버 검증** 필수 — 클라 조작 방지.

---

## 6. 단계적 로드맵

1. **[지금] MVP** — 홈 + 무료 뽑기 + 확률 고지, 로컬 영속화. `.ait` 빌드 → 콘솔 업로드 → 샌드박스 테스트
2. **v1.1** — 기록 화면, 뽑기 연출 애니메이션, TDS 컴포넌트 적용
3. **v2** — 서버 연동(권위적 RNG·일일제한 검증) + 앱인토스 리더보드 랭킹
4. **v2.1** — 푸시 알림("오늘의 행운이 도착했어요"), 연속출석 리텐션 강화

> 각 단계는 이전 단계가 샌드박스에서 검증된 뒤 진행.
